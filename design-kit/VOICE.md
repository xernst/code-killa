# Voice & Tone

Confident, sharp, no-hedge. The brand sounds like a senior engineer who already shipped what you're trying to learn.

## The voice in one paragraph

Promptdojo speaks the way a senior dev talks at 11 PM after a debugging session — direct, slightly tired, slightly funny, never preachy. It assumes the reader is smart and busy. It uses lowercase headlines because shouting is a waste of letters. It points at problems instead of soft-stepping around them.

## Do

- **All-lowercase headlines.** Always. Even product names sometimes (`promptdojo`, never `Promptdojo` in headline contexts).
- **Short sentences.** Cut anything that feels like throat-clearing.
- **Show the bug before naming it.** "ai writes this." then the broken code. The reveal is the hook.
- **Use "you."** Not "the learner" / "the user" / "students." You're talking to a person who clicked a link.
- **Borrow the audience's slang.** Vibe-coding, agentic, ship it, the wedge, the loop. If devs use the word, use the word.
- **Be specific about numbers.** "624 interactive steps," "ran in 187ms," "~100 lines of python." Specificity reads as confidence.

## Don't

- **No "as a developer..." preambles.** The reader is not a category.
- **No "we believe" / "we think."** State it. If you have to qualify, you didn't have an opinion.
- **No "amazing / incredible / game-changing."** These words mean nothing.
- **No emoji in copy.** Anywhere. The wordmark caret `❯` is not an emoji.
- **No exclamation points.** One per quarter, max, and only ironically.
- **No "Hey there!" intros.** The page is the intro.
- **No "stay tuned" / "soon" / "coming soon."** Either it's shipping or it isn't. If it isn't, say what changes when it does.

## Headline templates

| Template | Example |
| --- | --- |
| `[noun]. [contradiction].` | `ai writes this. it's wrong.` |
| `no [overhead]. no [overhead]. no [overhead].` | `no installs. no venv. no setup.` |
| `[verb] [outcome] in [n] [units].` | `ship a working cli agent in 12 steps.` |
| `[adjective].` | `forever.` (for the $0 reveal) |
| `[ironic understatement].` | `welcome to the dojo.` |

## Body-copy tone samples

**Right:**
> Promptdojo teaches you to ship working code in the AI era — not to memorize syntax. You'll review what the model writes, find the bug, fix it in-browser, and repeat 624 times.

**Wrong:**
> Welcome to Promptdojo, a comprehensive learning platform designed to empower modern developers in mastering Python through AI-assisted educational experiences. Our innovative curriculum combines cutting-edge techniques with hands-on practice.

## Punctuation rules

- **Em dashes** (`—`) for asides. Hard breaks, not soft.
- **Curly quotes** in body copy (`'` `'` `"` `"`), straight quotes in code samples only.
- **Lowercase chapter labels.** `chapter 07 · mutation and state` — lowercase + the centered dot, never `Chapter 7: Mutation and State`.
- **Tilde** (`~`) for approximations: `~100 lines of python`. Not "about 100" or "approximately 100."

## Off-limits topics

Per Josh's working agreement: **don't write public copy** about XWELL, biosurveillance, or HS-class governance angles unless explicitly greenlit. These are not the brand's themes.

## When in doubt

Read it out loud. If you sound like a marketing department, rewrite. If you sound like the person reading it would respect, ship it.
